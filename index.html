Language Trainer

Платформа для тренировки лексики с 8 форматами активностей и встроенным ИИ-агентом для персонализированных диалогов.

58учеников
2учителя
8форматов заданий
Что делала: Спроектировала и собрала веб-платформу для тренировки лексики с помощью вайбкодинга на Base44. Платформа включает восемь форматов активностей на каждую тему: флэшкарты, Matching, Квиз, Сортировка, Выбор фразы, Диалог, Погоня в лабиринте и Тайм-атака. Упражнение "Диалог" работает на базе встроенного ИИ-агента, который генерирует персонализированные диалоги из изучаемых слов. Учителя самостоятельно наполняют платформу: создают наборы слов, которые автоматически становятся полноценным "путём навыка" с восемью типами заданий.

Для кого: Для учителей иностранного языка, которые хотят давать интерактивные задания без лишних инструментов, и для учеников, которые занимаются в собственном темпе — с геймификацией, уровнями прогресса и выбором сложности.

Результат: Готовый к использованию тренажёр с системой прогрессии, геймифицированными механиками и встроенным ИИ — доступен по ссылке без установки и регистрации.

Ничего не горит

Веб-приложение для отслеживания учебных дедлайнов с дашбордом, календарём, системой приоритетов и двумя ролями.

4уровня приоритета
2роли
0установок
Что делала: Спроектировала и собрала веб-приложение для отслеживания учебных дедлайнов на Base44 с помощью вайбкодинга. Платформа агрегирует задания по всем предметам в одном месте: показывает дашборд с ближайшими и просроченными дедлайнами, поддерживает приоритеты (критический, высокий, средний, низкий), фильтрацию по предметам и группам, а также отдельный календарный вид. Предусмотрены две роли — администратор управляет дедлайнами и группами, студент видит свои задания.

Для кого: Для студентов учебных программ, у которых много параллельных предметов и заданий — чтобы не терять дедлайны и видеть полную картину нагрузки на неделю вперёд.

Результат: Работающий трекер с дашбордом, календарём, системой приоритетов и уведомлениями — доступен по ссылке без установки.

Конструктор промптов

Интерактивный HTML-конструктор для создания методически выверенных промптов к академическим текстам.

3этапа работы
0установок
∞дисциплин
Что делала: Разработала интерактивный HTML-конструктор промптов для проведения обучения преподавателей работе с ИИ. Инструмент проводит участника по трём этапам работы с академическим текстом — предтекстовому, текстовому и послетекстовому — и формирует готовый промпт под каждый этап. Конструктор учитывает дисциплину, уровень студентов, нужные умения академического чтения и требования формирующего оценивания. Отдельный модуль генерирует табличные критерии оценки с измеримыми дескрипторами.

Для кого: Для преподавателей на тренинге или воркшопе по использованию ИИ в учебном процессе — инструмент используется как раздаточный материал, который участники сразу применяют на практике: заполняют пропуски и получают готовый, методически выверенный промпт без необходимости разбираться в промптинге с нуля.

Результат: Практический инструмент в формате веб-страницы, который работает прямо на обучении: преподаватель заполняет пропуски, нажимает "скопировать" и вставляет промпт в любой ИИ. Без установки, без регистрации, сразу в деле.

Telegram-бот для образовательных мероприятий

Архитектура и проектная документация на ИИ-автоматизацию полного цикла коммуникации с участниками.

-77%трудозатрат
4331%ROI
25→55%заполняемость
Что делала: Спроектировала архитектуру Telegram-бота для автоматизации полного цикла коммуникации с участниками образовательного мероприятия: от онбординга и напоминаний до сбора обратной связи и рассылки материалов. Разработала три ключевых сценария автоматизации (адаптация участника, сопровождение событий, постпрограммное взаимодействие), модель данных, систему промптов для ИИ-слоя и блок этики и безопасности в соответствии с 152-ФЗ. Стек: Telegram Bot API + n8n + GigaChat API + DeepSeek API.

Для кого: Заказчик под NDA — краткосрочная интенсивная образовательная программа с аудиторией около 100 участников. Проблема: коммуникация велась вручную, оргкоманда тратила 3-4 часа в день на типовые запросы, заполняемость форм обратной связи не превышала 30%.

Результат: Готовая проектная документация на ИИ-автоматизацию: архитектура системы, сценарии с n8n-реализацией, модель данных, библиотека промптов и экономическое обоснование. По расчётам проекта: сокращение ручных трудозатрат команды на 77% (с 32 до 7 часов в неделю), рост заполняемости форм обратной связи с 25% до 55%, ускорение ответа на типовые вопросы с 2-24 часов до мгновенного. ROI по экономии трудозатрат — 4 331%.
